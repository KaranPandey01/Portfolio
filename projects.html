02 / HIGH-PERFORMANCE SYSTEMS
OrderBook — C++ Limit Order Book Engine
Jan 2026 – May 2026
C++17 pybind11 FastAPI React D3.js SQLite JWT Docker
GitHub →

A high-performance matching engine built in C++17 with price-time priority — bridged to a full Python/web stack via pybind11. Achieves 820K+ orders/sec sustained throughput. Features a portfolio/wallet system, real-time balance enforcement, and full transaction history.

Technical highlights
  • Built matching engine using std::map<int64_t, deque<Order>> with integer tick pricing to eliminate IEEE 754 floating-point correctness bugs present in naive implementations
  • Implemented O(1) order cancellation via unordered_map<order_id, deque::iterator> pointing directly into per-level FIFO queues; benchmarked 820K+ orders/sec at 1M order load
  • Bridged C++ engine to Python via pybind11; exposed 13 REST endpoints through FastAPI with JWT (HS256) auth, bcrypt password hashing, and SQLite persistence using WAL mode
  • Designed portfolio/wallet system with real-time balance enforcement, automatic refunds on cancellation, holdings tracking with average price calculation, and full transaction history
